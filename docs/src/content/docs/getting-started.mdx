This guide takes a new Managed Service Provider (MSP) from signup to a
working Conduit organization with one vendor integration connected and
an AI client making tool calls through it. Expect 20–40 minutes the
first time.

## Before you start

You need:

- An Auth0-backed login for your MSP (Conduit invites are sent by email
  during signup).
- An alpha invite code from your Conduit onboarding contact (unlocks
  the Pro plan; without it the MSP org starts on the free plan).
- API credentials for at least one vendor you want to connect (for
  example a ConnectWise Automate API key pair or a Datto RMM API key
  and secret).

:::note
All steps below are shown against the web UI. Every action also has an
API equivalent — see the [API reference](/docs/reference/api/).
:::

## 1. Create your MSP organization

1. Visit `https://<your-conduit-host>/signup` and complete the Auth0
   sign-in.
2. On first login, Conduit prompts you to create an **organization**.
   Enter your MSP's legal name; this is what customers see on invite
   emails and consent screens until branding is configured.
3. Paste your alpha invite code when prompted. This upgrades the org
   to the Pro plan immediately.

After creation you land on the dashboard.

## 2. Branding (optional)

White-label branding — the name, logo, and colors customers see — is
configured at the **deployment level** by whoever operates your
Conduit deployment. There is no self-service branding UI yet. See the
[white-label setup guide](/docs/guides/white-label-setup/) for the
environment variables that control it.

## 3. Connect your first vendor

A **vendor connection** is Conduit's adapter to a third-party vendor
API (ConnectWise, Datto RMM, IT Glue, and others). Connect it once at
the organization level.

1. Go to the **Connections** page (`/org/connections`).
2. Pick the vendor. API-key vendors show a credential form; OAuth
   vendors (Xero, QuickBooks, HubSpot, Microsoft 365) redirect to the
   vendor for consent.
3. Submit. Conduit encrypts the credential and runs a health check.
   A green status means the connection is ready.

For credential scopes, rotation, and OAuth specifics, see the
[vendor connections guide](/docs/guides/vendor-connections/).

## 4. Connect an AI client

Connect Claude Desktop or Claude Code to Conduit's MCP endpoint:

**Claude Desktop** (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "conduit": {
      "url": "https://<your-conduit-host>/v1/mcp"
    }
  }
}
```

**Claude Code**:

```bash
claude mcp add conduit https://<your-conduit-host>/v1/mcp
```

First use opens a browser for Auth0 sign-in. After sign-in the JWT is
cached and refreshed automatically.

## 5. Verify the setup

- Make a test tool call from Claude against one of the connected
  vendors.
- Open the **Audit log** in the dashboard and confirm the call is
  recorded with the correct user and vendor.
- Open the vendor health view and confirm every connected vendor
  shows green.
