# Implementation Workflow

1. Create or acknowledge a cortextOS task.
2. Inspect the repo before editing.
3. Define success criteria.
4. Create/confirm a feature branch.
5. Add characterization or failing tests when behavior changes.
6. Make scoped edits.
7. Run narrow tests, then broader relevant tests.
8. Update docs/templates if behavior changed.
9. Prepare a PR summary with tests and risks.
10. Never merge without explicit approval.

Use existing project patterns over new abstractions.
